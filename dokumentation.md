\# Dokumentation – Webbprojekt



\## Planering



Jag valde att skapa en informationssida om psykisk hälsa hos unga eftersom det är en aktuell och viktig samhällsfråga. Många ungdomar upplever stress och psykisk ohälsa och syftet med sidan är att informera och att öka förståelsen bland unga idag.



Målgruppen är ungdomar och elever, vilket påverkade hur jag utformade både innehåll och design. Jag valde att skriva tydligt och enkelt utan svåra begrepp så att informationen blir lätt att ta till sig.



Jag planerade webbplatsen med flera undersidor:

\- Startsida (introduktion)

\- Orsaker

\- Symptom

\- Hjälp/stöd

\- Källor



Jag gjorde en enkel layout med header, navigation, innehåll och footer. Menyn placerades högst upp för att göra det lätt att navigera mellan sidorna.



Designen är lugn och enkel med gröna och blå färger för att skapa en trygg känsla som passar ämnet.



\---



\## Genomförande



Jag började med att skapa mappstruktur och filer, inklusive HTML-sidor, CSS och bildmapp.



Därefter byggde jag upp grundstrukturen i HTML med semantiska element som <header>, <nav>, <main> och <footer>. Detta gjorde sidan mer strukturerad och tillgänglig.



Jag kopierade samma grundlayout till alla sidor för att få en enhetlig design. Sedan fyllde jag varje sida med relevant innehåll.



Efter det arbetade jag med CSS för att skapa designen. Jag använde flexbox i navigationen för att göra sidan responsiv och fungerande på både mobil och dator.



Jag lade även till:

\- Hover-effekter för bättre användarupplevelse

\- Responsiv design med media queries

\- Begränsad textbredd för bättre läsbarhet



Ett problem jag stötte på var att bilder blev för stora. Det löste jag genom att använda `max-width: 100%` i CSS.



Ett annat problem var att få navigationen att fungera bra på mobil, vilket jag löste genom att ändra layouten till kolumn vid mindre skärmar.



\---



\## Utvärdering



Jag tycker att resultatet blev bra. Sidan är tydlig, lätt att navigera och fungerar både på dator och mobil.



Det som gick bra var:

\- Att skapa en tydlig struktur

\- Att få designen lugn och enhetlig

\- Att göra sidan responsiv



Det som var svårare var:

\- Att förstå hur CSS påverkar layout

\- Att få alla delar att fungera tillsammans



Jag har lärt mig:

\- Hur man bygger en webbplats med HTML och CSS

\- Hur man arbetar med responsiv design

\- Vikten av tillgänglighet

\- Hur man planerar innan man börjar koda



Om jag skulle förbättra sidan skulle jag:

\- Lägga till mer interaktivitet med JavaScript

\- Utveckla designen ytterligare

\- Lägga till fler funktioner, t.ex. formulär
